How It Works

Five pipeline stages, zero partisan input.

Pipeline Overview

  1. Build precinct graph — Census TIGER 2020 VTDs become graph nodes; edge weights encode inverse road-network distance (highways penalise cuts across natural barriers)
  2. MCMC sampling — Weighted ReCom (DeFord, Duchin & Solomon 2021): at each step, select a random boundary edge, merge the two districts, sample a spanning tree via Wilson's loop-erased random walk with weights w−β, find a population-balanced cut
  3. Hard filters — keep only plans with Polsby-Popper minimum ≥ 0.05 and max population deviation ≤ 0.5%
  4. Pareto frontier — from up to 5,000 sampled plans compute county splits; select the non-dominated set across four objectives: maximise PP mean, minimise county splits, minimise cut edges, minimise max county fragmentation
  5. Extract two maps — most compact plan and fewest-splits plan from the frontier, both subject to minimising worst-case county fragmentation first
The algorithm never reads partisan registration, election results, race, income, or any demographic variable.
